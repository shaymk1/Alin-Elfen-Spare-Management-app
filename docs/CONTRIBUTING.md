# Contributing

Thanks for contributing! Please follow these simple guidelines to make reviews fast and consistent.

- Work on a feature branch created from `main`.
  - Branch name example: `feature/dashboard-ui` or `fix/login-typo`.
- Keep commits small and focused. Use imperative commit messages (e.g. `feat(ui): add sidebar`).
- Run tests (if available) and linting before opening a PR.
- Open a Pull Request and include a short description of the change and any testing notes.
- Maintain code style consistent with the project (`black`/`flake8` optional).
- For documentation or user-facing changes, update files in `docs/` as needed.

If you need to make a release or package the executable, see `docs/SETUP.md` for build steps.
