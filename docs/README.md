# Project Documentation

This `docs/` folder contains project documentation for the Spare Manager application. Use these documents to understand architecture, development setup, testing, release and security practices.

Included files

- `architecture.md` — Why this tech stack and packaging choices.
- `workflows.md` — Development and release workflows.
- `constraints.md`, `decisions.md`, `future-improvements.md`, `lessons-learned.md`, `ux-thinking.md` — project notes and design artifacts.
- `CONTRIBUTING.md` — How to contribute.
- `CHANGELOG.md` — High-level change log for releases.
- `SETUP.md` — How to set up a dev environment and build the executable.
- `SECURITY.md` — Notes on security and reporting.
- `TESTING.md` — How to run tests (if any) and test guidelines.
- `API.md` — Lightweight API / routes reference.

If you want additional documents here (design docs, runbooks, support), tell me which ones to add.
