# UX Thinking

## Goals

- Quick, at-a-glance information for store managers: inventory totals, low-stock alerts, active borrows and today's movements.
- Minimal cognitive load: reduce clicks to the most common tasks (view spares, view borrows, returns).
- Small-screen friendly: responsive layout that degrades gracefully for narrow displays.

## Dashboard

- Present four key metrics in a 2×2 grid for symmetry and easy scanning.
- Provide Quick Actions beside the metrics for common workflows.

## Navigation

- Top navigation: primary sections (Dashboard, Spares, Borrow, Return, Logout).
- Side panel: secondary/admin sections (History, Reports, Settings, Admin, Alerts) so admin features do not clutter the main nav.

## Accessibility

- Use high-contrast color variables and ensure labels are visible. Reserve color only for status (warning/success) rather than relying solely on color.

## Interaction Patterns

- Keep buttons full-width in cards where appropriate for touch-friendliness.
- Use simple modal confirmations for destructive actions.

## Visual Design

- Maintain consistent spacing, padding, and card elevation to create a clear visual hierarchy.
