(# Future Improvements

This list captures features and improvements that are out of scope for the initial offline executable but worth considering for future releases.

1. Authentication & Roles
	- Improve authentication with hashed passwords, role-based access control (fine-grained permissions), and optional integration with enterprise SSO for network deployments.

2. Import / Export
	- CSV/Excel import and export for spares, borrows and returns to aid migrations and bulk updates.

3. Reporting & Exports
	- Advanced reporting (PDF/Excel), scheduling, and filters for inventory, audit logs, and movement history.

4. Sync / Multi-user support
	- Optional synchronization layer for teams (sync server or peer-to-peer) to support concurrent edits and multi-user workflows.

5. Notifications & Alerts
	- Desktop notifications, low-stock automated alerts, and configurable thresholds.

6. Packaging & Platform Support
	- Provide signed builds for Windows, and native packages for macOS (signed) and Linux (.AppImage/.deb) with an automated build pipeline.

7. Auto-update mechanism
	- A secure auto-update channel so users can receive new versions without manual downloads.

8. Tests & CI
	- Add unit and integration tests, and set up CI to run tests and produce artifacts (built executables) automatically.

9. Accessibility
	- Improve color contrast, keyboard navigation, and screen-reader compatibility.

10. UX polishing
	 - More customizable dashboards, user preferences, and theming support.
